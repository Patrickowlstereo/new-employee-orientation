# 新人航行计划 · 项目优化工作总结

> 汇报对象：柔性小组
> 日期：2026 年 7 月 15 日
> 一句话概述：把原来"只能看、不能记"的演示原型，升级成了"能登录、能保存进度、能后台自助管理内容、能看全员学习统计、还有航海视觉体验"的完整可用系统。

---

## 一、这次优化做了什么（执行摘要）

原来的"新人航行计划"是一个**单页演示网页**——视觉效果好，但本质上是张"宣传画"：员工打开能看到地图和小岛，但**关掉浏览器一切归零**，没人知道谁学了什么、学到哪、有没有完成。

这次优化把它从"演示原型"升级为**真实可用的学习系统**，分六个阶段持续推进、均已交付：

- 员工用**自己的账号**登录，学习进度**自动保存**，下次打开接着学；
- 管理员能进入**独立的后台**，**自助上传/替换学习文档、维护机构与小岛、查看全员学习统计**，全程不用找开发改代码；
- 学习端迁移了原 Demo 的**航海视觉**（中国地图首页 + 海洋小岛），大视频**流式预览**不卡顿；
- 系统跑在**公司自己的服务器**上，数据存在公司自己的数据库里，不依赖外部服务。

六个阶段全部完成：①登录认证、②学习内容展示、③学习进度保存、④文档上传管理、⑤后台内容配置 + 全员统计、⑥航海视觉迁移 + 视频流式预览。

## 二、优化前 vs 优化后

| 维度 | 优化前（演示原型） | 优化后（可用系统） |
|------|------------------|------------------|
| 用户身份 | 没有，谁打开都一样 | 每人独立账号登录，识别身份 |
| 学习进度 | **不保存**，刷新即丢失 | **自动保存**，跨设备跨时间接着学 |
| 完成状态 | 看不出来 | 小岛/文档完成情况一目了然（已读、进行中、已完成） |
| 内容管理 | 改内容要改代码，找开发 | **管理员后台自助**上传/替换/删除文档、维护机构小岛，无需开发介入 |
| 学习监督 | 无 | **全员学习统计报表**：每人完成率、各小岛完成率、整体进度一目了然 |
| 文档预览 | 仅本地视频 | 图片/视频/音频/PDF 在线预览，**大视频流式播放可拖动进度条** |
| 视觉体验 | 精美但孤立 | **航海视觉已迁移**：中国地图首页 + 海洋小岛，与新系统打通 |
| 数据归属 | 仅存在于浏览器 | 存在公司数据库，可统计、可追溯 |
| 部署形态 | 一个网页文件 | 一个完整系统，可正式上线运行 |

## 三、新增的核心能力（业务价值）

### 1. 账号登录与身份识别
员工用"姓名全拼 + 密码"登录，系统识别每个人是谁。管理员和普通员工权限分开，管理员才能进后台。

### 2. 学习进度自动保存（核心改进）
员工每读一份文档、学到哪一页，系统都会**自动记录**：
- 哪些文档已读、哪些在读、哪些还没开始；
- 阅读进度百分比；
- 一个知识小岛下的必修文档全部学完，小岛自动标记为"已完成"。

员工中途离开、换台电脑、过几天再来，进度都在。这对**新人入职培训的到岗率与完成率**是直接支撑——培训负责人终于能知道"新人到底学没学完"。

### 3. 学习内容结构化展示
原有的 7 大机构（北京、上海、山东、四川、广东、重庆、浙江）、每个机构 5 个知识小岛、每个小岛的学习文档，全部结构化录入系统，员工按"机构 → 小岛 → 文档"的路径学习，路径清晰。

### 4. 管理后台（完整内容管理 + 统计）
管理员有**独立的后台入口**，用管理员账号登录。后台现已具备完整能力：
- **文档管理**：上传/替换/删除学习文档（见第 5 点）；
- **机构/小岛/文档行配置**：自助新增、改名、排序、停用，删除有保护（有小岛的机构、有文档的小岛、有学习记录的文档均拒删，引导改用"停用"）；
- **全员学习统计**：见第 6 点。

### 5. 文档上传管理（阶段 4）
管理员在后台**直接上传学习文档**，无需找开发改代码：
- 支持**全面文件类型**：文档（pdf/doc/docx/ppt/pptx/xls/xlsx/txt/csv）、图片（png/jpg/jpeg/gif/webp/bmp/svg/ico）、视频（mp4/webm/mov/mkv/avi/flv/m4v）、音频（mp3/wav/aac/m4a/ogg/flac）、压缩包（zip/rar/7z）；
- 上传即**记录审计**（上传时间、上传人），可追溯；
- 文件存于受保护目录，**路径穿越防护**，只能经鉴权接口访问；
- 学习端按类型在线预览（图片/视频/音频/PDF），其余类型提供下载。

### 6. 全员学习统计报表（阶段 5）
面向培训负责人与领导的**只读统计**，零图表库依赖：
- **个人维度**：每位员工的必修完成数、完成率、最后学习时间、小岛完成数；
- **整体维度**：总账号数、学员数、全部必修完成人数、平均完成率；
- **小岛维度**：各小岛的完成人数与完成率（进度条直观看板）。
统计在后端一次加载、内存批量计算，避免反复查库。

### 7. 航海视觉迁移 + 视频流式预览（阶段 6）
- **中国地图首页**：7 大机构作为浮动卡片按地理位置铺在地图上，点击进入该机构；
- **海洋小岛页**：机构下的知识小岛沿弧线排布于海面，岛间航线连接，按学习进度显示"锁定/解锁/已完成"，视觉与原 Demo 一致；
- **大视频流式预览**：学习资料的视频支持**拖动进度条、分片加载**，不再整文件载入内存，大视频不卡顿。

### 8. 一体化部署
整个系统——学习页面、管理后台、数据接口——打包成**一个程序**，部署在公司内网服务器上即可运行，无需复杂的多机部署。

## 四、质量与可靠性

我们对每一块功能都做了**分阶段开发 + 独立测试 + 交叉评审**，确保做出来的东西是真的能用、不是"看起来能用"：

- **分阶段、分小任务逐步交付**，每个任务独立可验收，出问题能快速定位；
- **关键功能都有自动化测试把关**，累计 **40 项后端自动化测试全部通过**（登录安全、密码加密、学习进度计算、文档上传、内容配置、统计、文件流式、接口鉴权等核心逻辑）；
- **每完成一步都经过独立代码评审**，发现问题当场修复，不留到后面；
- **整体完工后又做了全量复核**，发现并修复了多处可改进点（系统出错日志、文档缺失提示、前端网络异常、安全测试补充、进度上报节流、异常透传等），均已解决。

简单说：**不是赶工交差，而是边做边验、做完又复查**。

## 五、安全性

面向公司内部系统，安全是底线。本次已落实：

- **密码加密存储**：员工密码不存明文，采用业界标准的加密方式（即使数据库被拷走也看不到原始密码）；
- **登录防探测**：输错账号或密码，系统统一提示"账号或密码错误"，不暴露哪个账号存在，防止被人试探；
- **登录凭证有时效**：登录后发放的"通行证"有时限，过期需重新登录，避免账号被他人盗用长时间；
- **后台权限隔离**：普通员工无法访问管理后台接口，管理员才能操作；
- **文档下载需登录**：学习资料不能被未登录的人直接下载链接拿走；
- **上传文件类型受限**：只允许白名单内的文档/图片/视频/音频/压缩包，挡住可执行脚本等危险文件；
- **文件存储防穿越**：上传文件存于受保护目录，校验路径不得越界，防止恶意路径读取服务器其他文件；
- **媒体鉴权最小化**：为支持视频流式预览，仅文件下载接口允许登录态通过链接参数传递，其余接口一律走标准令牌头；
- **数据存公司内网**：所有数据存在公司自己的数据库服务器，不外传。

## 六、采用的技术（通俗说明）

为让领导放心，所采用的都是**业界主流、成熟、好招人维护**的技术，不存在"小众技术踩坑"风险：

- 后端用 **Java**（金融国企主流选择，稳定、生态成熟）；
- 数据库用 **PostgreSQL**（开源关系型数据库，免费、可靠）；
- 前端用 **React**（全球最主流的界面技术，人才储备充足）；
- 大视频采用 **HTTP Range 分片流式**（业界标准做法，浏览器原生支持，无需额外流媒体服务器）；
- 部署形态：**公司内网裸机直接运行**，无需云服务费用。

## 七、后续规划

六个阶段已全部完成，系统具备完整的学习 + 管理 + 统计 + 视觉能力。剩余优化项建议按需推进：

| 事项 | 内容 | 业务价值 |
|------|------|---------|
| 员工账号管理 / SSO | 对接公司统一认证，或后台自助创建/停用/重置账号 | 免去人工建号，账号生命周期可控 |
| 图片资源压缩 | 机构城市图原图各约 4MB，首页首载偏重 | 首页打开更快，体验更顺滑 |
| 视频凭证续期体验 | 登录态过期时视频自动提示重登（当前需手动刷新） | 长视频学习中途不掉线 |
| 视觉进一步打磨 | 更多原 Demo 动效（海面光纹、涌浪等）按需迁移 | 视觉更精致（非必需） |

**几点说明**：

- 上述事项互不阻塞，可按业务节奏排期。
- 接入公司统一认证（SSO）取决于公司 IT 配合度与接口文档，约 3–5 个工作日。
- 当前员工账号为只读展示，正式开放账号管理前，可由运维通过数据库或脚本协助建号。
